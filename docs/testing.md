# Testing

## Backend
- unit (middlewares e regras)
- integration (HTTP)
- e2e (fluxo minimo de pedido)

## Frontend
- testes unitarios basicos de componentes/fluxos

## Regra de cobertura
- toda nova feature e todo novo caso de uso, tanto no frontend quanto no backend, devem vir acompanhados de testes com cobertura maxima

## Infra
- validacao de YAML/JSON no CI
